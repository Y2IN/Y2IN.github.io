# Next.js란?

Next.js는 React 라이브러리의 프레임워크이다. React에서 서버 사이드 렌더링을 간편하게 해주는 프레임워크이다.

[공식 홈페이지](https://nextjs.org/learn/basics/create-nextjs-app)에서 learn course를 제공하고 있는데 learn course만한 글이 없는 것 같다.

## Next.js를 통해 얻는 가장 큰 장점은?

요약 : javascript로 모든 페이지를 구성하는 React와는 다르게, Pre-rendering을 하지 않고 서버사이드 렌더링을 하게되면 검색엔진들이 html을 직접검색할 수 있으므로, 검색어 노출에 좋다. (SEO)

![Alt text](https://github.com/Y2IN/Y2IN.github.io/blob/main/public/static/images/blog_project/pre-rendering.png?raw=true)
기본적으로 React는 Client Side Rendering(CSR)을 한다. 이는 위 그림처럼 웹사이트를 요청했을 때 빈 html을 가져와 script를 로딩하기 때문에, 첫 로딩 시간도 오래걸리고 Search Engine Optimization(SEO)에 취약하다는 단점이 있다.
![Alt text](https://github.com/Y2IN/Y2IN.github.io/blob/main/public/static/images/blog_project/pre-rendering2.png?raw=true)
next.js는 pre-reloading을 통해 미리 데이터가 렌더링된 페이지를 가져올수 있게 해주므로 사용자에게 더 좋은 경험을 주고, 검색 엔진에 잘 노출 될 수 있도록 해주는 SEO에서도 장점을 얻을 수 있다.

# Next.js 프로젝트 시작 (create-next-app)

기본적으로 next.js를 사용하기 위해서는 node.js 설치가 필요한데, node.js는 간편하게 설치할 수 있다.

```zsh
npx create-next-app nextjs-blog
// or - typescript
npx create-next-app@latest --ts
// or
yarn create next-app --typescript
// or
pnpm create next-app -- --ts
```

# Next 서버 실행하기

```zsh
npm run build

// 프로젝트 폴더로 들어가서
npm run dev
```
위의 명령어를 통해 로컬 환경에서 웹 페이지를 띄워볼 수 있다.
![Alt text](https://github.com/Y2IN/Y2IN.github.io/blob/main/public/static/images/blog_project/s1.png?raw=true)

여기서 https://localhost:3000 링크로 들어가면 위와 같은 화면이 뜬다.

이 화면이 뜨면 로컬 환경에서 웹 페이지가 제대로 실행되고 있다는 뜻이다.
